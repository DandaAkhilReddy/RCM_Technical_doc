Data Flow Analysis

Complete patient encounter data journey from admission through payment posting

Patient Encounter Data Journey

Day 0 - Real-time
1. Patient Admission (Meditech)

Data Created: Demographics, Insurance, ADT events, Room/Bed assignments

System: Hospital EHR (Meditech)

Status: Live in clinical system

Day 1 - 24 Hour Delay
2. Census Export (Meditech to IngeniousMed)

Process: Daily file export at midnight

Format: CSV/Text file

Critical Issue: 24-hour lag creates duplicate entry risk

Day 1-2
3. Charge Capture (IngeniousMed)

Data Added: Provider rounds, CPT codes, ICD-10 diagnoses, modifiers

Manual Step: Providers document daily visits and procedures

Validation: Coders review for accuracy and compliance

Day 2-3
4. Billing Transfer (IngeniousMed to AdvancedMD)

Integration: API-based daily scheduled sync

Data Transferred: Patient demographics, insurance, charges, diagnoses

Status: Automated but requires validation

Day 3-5
5. Claim Creation (AdvancedMD)

Process: Scrubbing, validation, batching

Format: 837 Professional EDI

Output: Electronic claims to clearinghouse

Day 5-7
6. Claim Submission (McKesson RelayHealth)

Clearinghouse Processing: Format validation, payer routing

Status Updates: Acceptance/rejection notifications

Tracking: Real-time claim status monitoring

Day 14-45
7. Remittance Receipt (835 ERA)

Return Path: McKesson to AdvancedMD

Data Received: Payment amounts, adjustments, denial codes

Auto-posting: ERA auto-posts to patient accounts

Day 14-60
8. Payment Reconciliation (Manual)

Critical Gap: No bank integration

Process: Manual Excel tracking of bank deposits

Matching: ERA amounts vs actual deposits (manual)

This is the primary bottleneck causing revenue leakage

Data Transformation & Enrichment

Stage 1: Clinical Data (Meditech)

Data Elements
Patient Name, MRN, DOB, Insurance, Admission Date
Format
HL7 ADT Messages (internal)
Export Format
CSV/Text File (to IngeniousMed)
Frequency
Daily at midnight

Stage 2: Charge Capture (IngeniousMed)

Data Added
CPT codes, ICD-10, Modifiers, Units, Provider NPI
Enrichment
Clinical documentation, compliance checks
Validation
Medical coding rules, insurance requirements
Output Format
API JSON payload

Stage 3: Billing System (AdvancedMD)

Data Processing
Fee schedule application, claim generation
Scrubbing
Duplicate checks, eligibility verification
Output Format
837 Professional EDI
Transmission
Automated to clearinghouse

Stage 4: Payment Processing (Manual)

ERA Receipt
835 EDI auto-imported
Bank Deposit
Manual download to Excel
Matching
Manual ERA-to-deposit reconciliation
Posting
95% manual entry required

Current Automation Status

Process Step Current Method Automation Level Time Required Error Rate
Meditech to IngeniousMed Daily file export Automated Overnight batch Low (duplicates occur)
Charge Capture Provider documentation Manual 15-30 min/provider/day Medium (coding errors)
IngeniousMed to AdvancedMD API scheduled sync Automated Daily scheduled Low
Claim Scrubbing AdvancedMD built-in Semi-Auto 2-4 hours/day Medium (requires review)
Claim Submission EDI to clearinghouse Automated Real-time Very Low
ERA Receipt 835 EDI auto-import Automated Real-time Low
Bank Reconciliation Manual Excel tracking Manual 10-15 hours/week High (missed payments)
Payment Posting Manual entry Manual 20-25 hours/week High (data entry errors)
Critical Finding: The final mile of the revenue cycle (bank reconciliation and payment posting) is 95% manual despite receiving automated ERA data. This creates a significant bottleneck and prevents real-time visibility into actual cash received.
