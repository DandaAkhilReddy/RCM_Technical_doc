Integration Methods

Detailed analysis of system integration methods, protocols, and connectivity

Integration Methods Overview

From System To System Method Frequency Status Notes
Meditech IngeniousMed File Export Daily (24hr lag) ⚠️ Active Causes duplicate entries, manual reconciliation needed
IngeniousMed AdvancedMD API Daily scheduled ✅ Functional Automated charge import, minimal issues
AdvancedMD McKesson 837 EDI Real-time ✅ Functional Standard claims submission process
McKesson AdvancedMD 835 ERA As received ✅ Functional Electronic remittance advice auto-posting
Bank AdvancedMD NONE Manual Excel ❌ Critical Gap 100% manual reconciliation required
AdvancedMD EnableComp SFTP 120+ days AR ⚠️ Limited One-way transfer, no visibility into status
⚠️ Critical Limitation: AdvancedMD does NOT support HL7 format, severely limiting integration capabilities with clinical systems. This forces reliance on daily file exports with 24-hour delays and prevents real-time data synchronization.

Integration Method Details

File-Based Integration (Meditech → IngeniousMed)

  • Format: Proprietary file export from Meditech
  • Frequency: Daily export, typically processed overnight
  • Issues: 24-hour delay causes duplicate charges, requires manual cleanup
  • Data: ADT (Admission/Discharge/Transfer) information, demographics

API Integration (IngeniousMed → AdvancedMD)

  • Protocol: REST API with OAuth authentication
  • Frequency: Daily scheduled synchronization
  • Reliability: High - minimal failures reported
  • Data: Professional charges, CPT/ICD codes, provider information

EDI Integration (AdvancedMD ↔ McKesson)

  • Standards: 837P (professional claims), 835 (ERA), 277 (claim status)
  • Frequency: Real-time claim submission, ERA as received from payers
  • Performance: Excellent - industry standard protocols
  • Coverage: All major payers supported
Recommendation: Implement middleware platform (Mirth Connect, Rhapsody, or custom solution) to enable HL7 integration capabilities and reduce dependency on file-based transfers.
